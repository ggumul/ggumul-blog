# ggumul-blog 공개 문장 검증 원칙

블로그 글과 프로젝트 카드는 실제 화면으로 확인한 범위만 말한다.

## 현재 확인된 근거

- `public/media/devlog-gifs/wanderer-rule-result.gif`
  - 말할 수 있음: `11 이하` 조건, `5/10/11` 선택 가능, `13/14/15` 선택 불가, 밝기 차이.
  - 말하면 안 됨: 전체 한 판 완료, 복귀 성공, 결과 화면, 버림 더미 흐름.
- `public/media/devlog-gifs/hanoi-next-seat.gif`
  - 말할 수 있음: Hanoi 화면, 원반 세 개, 세 기둥, 이동 횟수/원반 위치.
  - 말하면 안 됨: `다음 자리가 열린다`, `막힌 자리가 보인다`, 이동 가능/불가능 표시가 이미 있다.
- Color Hanoi / TRPG / Dinner Grocery
  - 현재 공개 글에서 쓸 수 있는 실제 GIF 근거 없음.
  - 기준과 다음 캡처 대상은 쓸 수 있지만, 이미 동작하는 화면처럼 말하지 않는다.

## 공개 문장 금지 패턴

- 화면에 없는 상태를 단정하지 않는다.
- `실제 GIF 없음`처럼 내부 결핍을 카드 문구로 노출하지 않는다.
- `검증`, `테스트`, `로컬`, `Simulator`, `확인했습니다`를 첫 화면이나 요약의 매력 포인트로 쓰지 않는다.
- Notion 초고의 관찰 로그를 그대로 복사하지 않는다.

## 글 작성 기준

1. Notion 초고를 먼저 쓴다.
2. 사용할 이미지/GIF/영상에서 실제로 보이는 문장과 숫자를 적는다.
3. 보이는 것 / 보이지 않는 것 / 아직 잡아야 할 화면을 분리한다.
4. MDX에는 보이는 것만 공개 주장으로 남긴다.
5. `npm test`로 과장 표현 회귀를 막는다.
