# ggumul-blog public reading reset

## 왜 다시 갈아엎는가

지금 사이트는 문장이 짧아졌지만, 사람이 읽는 흐름은 아직 무너져 있다. 가장 큰 문제는 꼬물의 실제 의미를 말하지 않고, `고르면 바로 결과가 보이는 게임`, `게임 뒤에 읽는 글`, `선택과 결과`라는 임의의 프레임으로 모든 페이지를 묶은 것이다.

꼬물의 기준은 다음 한 줄이다.

> 느리더라도 꾸준히 꾸물거리며 개발을 나아간다.

Notion과 예전 기획 문서에도 같은 뜻이 남아 있다. 다만 그 문장을 그대로 public UI에 복붙하면 또 딱딱한 소개문이 된다. public copy는 의미만 가져오고, 방문자가 첫 화면에서 읽을 수 있는 문장으로 다시 써야 한다.

## 1차 audit

| 위치 | 현재 문제 | 살릴 의미 | 새 방향 |
| --- | --- | --- | --- |
| 홈 | `고르면 바로 결과가 보이는 게임`이 꼬물 전체 철학처럼 보임. Wanderer/Hanoi/TRPG의 조작 예시가 hero에 섞임. | 꼬물은 작은 게임과 작업 기록을 오래 쌓는 곳. | “우리는 느리지만 멈추지 않고 게임을 만든다”를 사람 말로 쓰고, 방문자가 바로 볼 수 있는 것은 게임/글/작업 흔적이라고 안내. |
| 공통 header/footer | `작은 게임, 그리고 게임 뒤에 읽는 글`이 모든 페이지를 좁은 게임 설명으로 고정. | 사이트는 작업실/기록장. | `게임을 만들고, 만드는 과정을 적는 작은 작업실` 정도의 범위로 확장. |
| /projects | 모든 항목이 `게임`으로 묶여 Dinner Grocery 같은 도구도 어색하게 게임처럼 읽힘. | Wanderer 등 게임, 그리고 곁가지 도구가 함께 쌓임. | “만들고 있는 것들”로 풀고 게임/도구를 분리해서 읽히게 함. |
| /writing | `게임 뒤에 읽는 글` 반복. 글이 독립적인 읽을거리가 아니라 부속 설명처럼 보임. | 글은 개발 과정과 판단을 남기는 곳. | “만들면서 남긴 글”로 바꾸고, 제목/요약을 사람 읽는 문장으로 정리. |
| /about | `짧게 끝나는 작은 게임`으로 시작해서 꼬물 의미가 빠짐. | 느림, 꾸준함, 꾸물거림, 계속 만듦. | 이름의 뜻과 작업 태도를 먼저 말하고, 게임은 그 태도가 드러나는 결과로 둠. |
| /links | Notion/GitHub/메일 설명이 중복되고, `게임에서 시작해 코드, 아이디어...`가 억지 경로처럼 읽힘. | 바깥 링크를 단순히 정리. | GitHub/Notion/메일 각각을 필요한 순간에 맞게 짧게 설명. |
| 글/프로젝트 MDX | `선택/결과/흐름/상태/기준/정리`가 반복되어 내부 보고서처럼 보임. | 각 게임에서 실제로 중요한 조작과 고민. | 제목은 독자가 궁금해할 문장으로, 본문은 “왜 만들었나 → 무엇을 바꿨나 → 지금 어디까지 왔나” 순서로 간단히. |

## 새 정보 구조

1. 홈
   - 꼬물 정체성: 느리지만 멈추지 않고 만드는 작은 게임 작업실.
   - 바로 볼 수 있는 것: Wanderer 한 판, 만들고 있는 것들, 최근 글.
   - 게임과 글의 관계: 글은 게임의 부속 설명이 아니라 만드는 과정이 남은 글.
2. 만들고 있는 것들(`/projects`)
   - 게임: Wanderer, Hanoi, Color Hanoi, TRPG.
   - 도구: Dinner Grocery.
   - 각 항목은 “무엇인지 / 지금 어디까지인지 / 왜 남겨두는지”만 말함.
3. 글(`/writing`)
   - “게임 뒤의 글” 대신 “만들면서 남긴 글”.
   - 글 목록은 날짜 + 제목 + 한 문장만. 태그 pill 과잉 제거.
4. 소개(`/about`)
   - 꼬물 이름의 뜻.
   - 느림을 변명으로 쓰지 않고, 계속 손대는 태도로 설명.
5. 링크(`/links`)
   - GitHub, Notion, mail만 필요할 때 쓰는 문장으로 정리.

## 문장 원칙

- 꼬물 철학을 invent하지 않는다.
- `선택/결과/흐름/상태/기준/지점/정리/기록` 같은 추상어를 문장 중심에 놓지 않는다.
- `게임 뒤에 읽는 글`, `고르면 바로 결과`, `작은 선택`, `큰 세계관보다` 프레임을 버린다.
- Notion 문장은 source-of-truth이지 public copy가 아니다.
- CTA를 늘리지 않는다. 링크는 제목 자체로 충분히 이동 가능해야 한다.
- `검증`, `Simulator`, `Flutter`, `sync`, `로컬 서버` 같은 내부 구현 단어는 public archive 목록/hero에서 빼고, 기술 글 본문에서도 꼭 필요할 때만 쓴다.
- `만듭니다/이어집니다/보입니다`가 반복되면 보고서처럼 보이므로 문장 리듬을 섞는다.
