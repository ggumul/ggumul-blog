# ggumul full redesign

> 목표: 지금의 "예쁜 블로그"를 버리고, 꼬물의 작업 리듬과 축적 방식을 보여주는 아카이브형 작업실로 재설계한다.

## 왜 리셋해야 하나
- 현재 화면은 브랜드 철학을 설명하는 비중이 너무 크다.
- 카드와 요약 박스가 많아 작업실보다 템플릿 랜딩처럼 보인다.
- 꼬물의 핵심인 느린 축적, 발자국, 진행 중 상태가 구조보다 문장으로만 존재한다.

## 절대 안 잃을 것
1. 꼬물은 느린 팀이 아니라 오래 쌓는 팀이다.
2. 완성본보다 흔적과 발자국을 먼저 보여준다.
3. 프로젝트는 결과물이 아니라 붙들고 있는 감각의 묶음이다.
4. 글은 발표문보다 작업 노트에 가깝다.
5. 작은 작업실의 솔직함을 유지한다.

## 버릴 것
- NOTE 01/02/03 같은 설명 카드
- KPI처럼 보이는 숫자 요약 박스
- 운영 원칙 3열 소개 블록
- 브랜드 철학 반복 설명 섹션
- CMS 기본 카드처럼 읽히는 반복 박스 구조

## 새 구조 원칙
### 홈
- 첫 장면은 소개보다 최근 흔적이 앞에 온다.
- "가장 최근 기록" + "지금 붙든 작업" + "축적된 글 흐름" 순서.
- 설명은 짧게, 실제 기록과 연결을 길게.

### 글 목록
- 카드 진열보다 기록 축적감이 먼저 보이게 한다.
- 표지 비주얼은 보조, 제목/날짜/시리즈가 흐름을 만든다.
- 태그는 장식이 아니라 분류 체계로 다룬다.

### 프로젝트 목록
- 쇼케이스가 아니라 작업선(workline)처럼 보이게 한다.
- 순서, 상태, 관련 기록 연결이 구조적으로 드러나야 한다.

### 시각 문법
- 따뜻하지만 안전한 SaaS 톤 금지.
- 큰 라운드 박스 남발 금지.
- 여백은 넉넉하되, 정보는 박스로 쪼개지지 않게 한다.
- 꼬물만의 표기 규칙(기록, 발자국, 작업 중, 시리즈)을 전면 사용한다.

## 구현 기준
- 레퍼런스는 Panic의 아카이브 흐름, ustwo의 정돈된 콘텐츠 문법만 참고.
- 레퍼런스의 표면이 아니라 꼬물의 정서를 우선한다.
- 설명보다 흔적, 소개보다 축적, 발표보다 메모.

## 완료 기준
- 첫 화면만 봐도 "브랜드 소개 페이지"가 아니라 "작업실 아카이브"처럼 느껴질 것.
- 카드가 아니라 흐름이 먼저 읽힐 것.
- 꼬물의 말투와 감각이 레이아웃에서 체감될 것.
