이번에는 실제 화면에서 선택 뒤 결과가 바로 읽히는지 봤습니다. Wanderer는 카드 한 장을 고른 뒤 승부가 늦게 보이면 한 판의 리듬이 바로 흐려집니다.

처음에는 “앱이 켜진다”만 확인하면 충분해 보였습니다. 그런데 실제 화면을 이어서 보니 버튼을 누른 뒤 다음 상황이 바로 이해되지 않는 구간이 있었고, 한 판의 리듬도 생각보다 자주 끊겼습니다. 이 기록은 그 지점들을 이어서 볼 장면의 기준으로 남긴 글입니다.

화면이 없는 `wanderer-was` 쪽 작업은 이번 글에서 영상 대신 결과 확인만 남겼습니다. 이 기록은 플레이어가 직접 마주하는 화면 흐름에 초점을 둡니다.

## 먼저 본 것

- Hanoi: 로컬 브라우저에서 실행 후 퍼즐 조작 보기
- Wanderer 모바일: Flutter 환경 파일 생성 후 iOS Simulator에서 앱 실행 보기
- Wanderer: 빠른 대전에서 카드 제출과 결과 화면까지 보기

이번에는 플레이어가 실제로 마주하는 화면까지 자연스럽게 이어지는지를 먼저 봤습니다. 그래서 화면이 없는 서버 작업은 실행 영상과 분리했고, 화면이 있는 프로젝트만 영상으로 남겼습니다.

## Hanoi

Hanoi는 원반을 선택하고 다른 기둥으로 옮기는 기본 조작을 봤습니다. 중요한 건 조작 뒤 다음 상태가 한눈에 들어오는지였습니다.

현재 화면에서는 퍼즐 상태, 이동 횟수, 조작 버튼이 한 화면에 들어옵니다. 아직 더 봐야 하는 부분은 난이도 선택 이후의 플레이 흐름과 폰에서의 손 조작감입니다. 여기서는 앱을 열고 기본 조작을 따라가며, 다음 상태가 눈에 잘 들어오는지 봤습니다.

## Wanderer 모바일

Wanderer는 홈에서 빠른 대전으로 들어가고, 매칭을 기다린 뒤 카드 제출과 결과 화면까지 이어서 봤습니다. 짧은 상태 컷이 아니라 한 판의 흐름이 어디서 끊기거나 어색해지는지 함께 보려는 기록이었습니다.

영상 초반에는 홈/로비에서 빠른 대전을 시작하는 화면이 나오고, 이어서 매칭 대기, 카드 제출, 결과 화면까지 이어집니다. 그래서 단일 프레임이 아니라 한 판이 어디까지 이어지는지 볼 수 있습니다.

## 화면에서 먼저 본 이유

화면에서는 승부가 바로 읽히지 않으면 플레이 감각이 끊깁니다. 그래서 이 글은 카드 선택 뒤 장면이 어떻게 보이는지에 초점을 둡니다.

그래서 여기서는 사용자가 직접 볼 수 있는 화면을 먼저 남겼습니다. 카드 선택 뒤 승부가 바로 보이는지가 이 글의 중심입니다.

## 지금 기준의 결론

Hanoi는 기본 조작 뒤 다음 상태가 보이는지 봤습니다. Wanderer는 홈에서 빠른 대전으로 들어가 카드 제출과 결과 화면까지 한 흐름으로 봤습니다.

다음에는 같은 한 판이 더 오래 이어져도 카드 선택과 결과가 흔들리지 않는지 볼 예정입니다. Hanoi는 폰 화면에서 이동 결과가 바로 읽히는지도 따로 봅니다.
